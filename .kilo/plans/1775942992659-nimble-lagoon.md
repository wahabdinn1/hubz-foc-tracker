# Hubz FOC Tracker — Project Analysis

## Project Overview

**Hubz FOC Tracker** is an internal analytics dashboard and logistics tracking portal for managing Free-of-Charge (FOC) device loans. It aggregates device check-ins/outs and return schedules from Google Sheets into a unified Next.js web interface for Key Opinion Leader (KOL) operations.

| Layer | Technology |
|---|---|
| Framework | Next.js 16 (App Router, React 19) |
| Styling | Tailwind CSS v4, Shadcn UI, Framer Motion |
| State | `next-themes`, React Hooks, `useTransition` |
| Validation | Zod, react-hook-form |
| Data Source | Google Sheets API (via Server Actions) |
| Auth | `jose` (JWT), HTTP-only cookies, PIN-based |
| Package Manager | pnpm |

## Critical Issues (Must Fix)

### 1. Disconnected Middleware — No Route-Level Auth Guard
**File**: `src/proxy.ts` (entire file is dead code)

The file contains Next.js middleware logic with a `config` matcher, but it's named `proxy.ts` instead of `middleware.ts`. Next.js requires the file to be named `middleware.ts` at the project root or `src/` root. This means:
- The auth guard that sets `x-middleware-auth` headers **never executes**
- All page routes are accessible without auth at the routing level
- Auth checking only happens inside Server Components after the page loads
- **Fix**: Rename `src/proxy.ts` → `src/middleware.ts`

### 2. JWT Signing Key Falls Back to `GOOGLE_PRIVATE_KEY`
**Files**: `src/server/auth.ts:37`, `src/lib/auth.ts:12`, `src/proxy.ts:13`

```ts
const key = process.env.JWT_SECRET || process.env.GOOGLE_PRIVATE_KEY;
```

If `JWT_SECRET` is not set, the Google API private key is used as a JWT secret. This means:
- A credential meant for API authentication is dual-purposed for user auth
- If `GOOGLE_PRIVATE_KEY` is compromised, the attacker can forge auth tokens
- **Fix**: Make `JWT_SECRET` required. Remove the fallback.

### 3. Global Rate Limit Key — One Attacker Can Lock Out All Users
**File**: `src/server/auth.ts:25`

```ts
const rateLimitKey = "pin-global";
```

Uses a single global key instead of per-IP or per-session. After 5 global failures, ALL users are locked out for 15 minutes. An attacker can intentionally lock out all users with just 5 rapid failed attempts.
- **Fix**: Use request IP or session identifier as the rate limit key

### 4. Transfer Operation is Not Atomic
**File**: `src/server/mutations.ts:507-558`

The transfer writes to two sheets sequentially (return from KOL 1, then issue to KOL 2). If the first write succeeds but the second fails, the system is in an inconsistent state — the device is recorded as returned but never re-issued.
- **Fix**: Add a compensating transaction or manual reconciliation flag

## High-Priority Issues

### 5. No Input Sanitization Before Writing to Google Sheets
**File**: `src/server/mutations.ts:262-278` (and all write operations)

Form data is written directly without output sanitization. A malicious input like `=HYPERLINK(...)` or `=IMPORTDATA(...)` could be interpreted as a spreadsheet formula (formula injection).
- **Fix**: Prefix cell values with `'` (apostrophe) for strings starting with `=`, `+`, `-`, or `@`

### 6. Race Condition in `writeToNextRow` (TOCTOU)
**File**: `src/server/mutations.ts:76-101`

Between reading the current row count and writing to the next row, a concurrent request could write to the same row. While `requestUnit` has collision detection, `returnUnit` and `transferUnit` do not.
- **Fix**: Use `INSERT_ROWS` append mode instead of read-then-write pattern, or add collision detection to all mutation paths

### 7. In-Memory Rate Limiting Not Serverless-Safe
**File**: `src/lib/rate-limit.ts:20`

Uses an in-memory `Map` that resets on every cold start in serverless environments (Vercel).
- **Fix**: Use an external store like Upstash Redis or Vercel KV

### 8. Accessibility — Interactive Elements Without Keyboard Support
**Files**: `ReturnTrackingTable.tsx:64-66`, `KOLClient.tsx:118-124`, `KOLClient.tsx:206-214`

Multiple clickable `div`/`motion.div` elements act as buttons but lack `tabIndex`, `role`, or `onKeyDown` handlers. Screen reader and keyboard users cannot access these interactive elements.
- **Fix**: Add `role="button"`, `tabIndex={0}`, and `onKeyDown` handlers, or use semantic `<button>` elements

## Medium-Priority Issues

### 9. Duplicated Helper Functions Across Components
**Files**: `RequestFormModal.tsx`, `TransferFormModal.tsx`, `RequestFormDevice.tsx`

`getDeviceCategory()`, `extractFocType()`, `getCategoryIcon()`, `FOC_TYPE_KEYS` are duplicated across 2-3 files.
- **Fix**: Extract to `src/lib/device-utils.ts`

### 10. `useInventoryStats` Does 6+ Separate Filter Passes
**File**: `src/hooks/useInventoryStats.ts:27-157`

Six separate `.filter()` calls over the inventory array. A single pass could compute all counts and arrays simultaneously.
- **Fix**: Consolidate into a single `useMemo` that produces all derived values

### 11. `topUrgentReturns` Sort Uses `new Date()` Without Format-Aware Parsing
**File**: `src/hooks/useInventoryStats.ts:101-103`

```ts
const dateA = new Date(a.plannedReturnDate).getTime();
```

When `plannedReturnDate` is a non-standard format, `new Date()` may return `NaN`, making `.sort()` behavior undefined. A `parseDateStr` helper exists but isn't used here.
- **Fix**: Use `parseDateStr()` for consistent date parsing

### 12. `giftedUnitsCount` Uses Different Base Array (Bug)
**File**: `src/hooks/useInventoryStats.ts:41-43`

Uses `inventory` (unfiltered) while `availableCount` and `onKolCount` use `validInventory` (filtered for non-empty). This count could include empty-row items.
- **Fix**: Change to filter from `validInventory`

### 13. AuditLogTable `allEvents` Not Memoized
**File**: `src/components/audit/AuditLogTable.tsx:102-138`

Array transformation runs on every render without `useMemo`, creating unnecessary GC pressure.
- **Fix**: Wrap in `useMemo`

### 14. Import After Statement
**File**: `src/server/inventory.ts:312`

An import statement appears in the middle of the file, after a function's closing brace.
- **Fix**: Move import to top of file

### 15. PINs Stored in Plain Text
**File**: `src/server/auth.ts:34-36`

PINs are stored as a comma-separated string and compared with a direct string match. No hashing or timing-safe comparison.
- **Fix**: Use `crypto.timingSafeEqual` or hash PINs with bcrypt

### 16. Mobile Menu Button Missing ARIA Attributes
**File**: `src/components/layout/DashboardLayout.tsx:67`

No `aria-label`, `aria-expanded`, or `aria-controls` attributes on the mobile menu button.
- **Fix**: Add appropriate ARIA attributes

### 17. Donut Chart Not Accessible
**File**: `src/components/dashboard/DashboardDonutChart.tsx:56-87`

SVG donut chart has no `role="img"`, `aria-label`, or `<title>` element.
- **Fix**: Add `role="img"` and `aria-label` with chart description

## Architecture Strengths

- **Well-structured Server Actions pattern** with barrel export, focused modules, and proper `"use server"` directives
- **Dual auth check** — Server Components AND Server Actions both verify authentication
- **Smart column mapping** — Header-name matching with fallback to positional indices makes the app robust against sheet changes
- **Centralized constants** — All dropdown values, sheet names, and column headers in `src/lib/constants.ts`
- **Proper form decomposition** — Request form broken into `RequestFormCampaign`, `RequestFormDevice`, `RequestFormKol`, `RequestFormDelivery` sub-components using `useFormContext()`
- **Client/server schema split** — Zod validation correctly separates client schemas (using `z.date()`) from server payload schemas (using `z.string().regex()`)
- **Race-condition detection in `requestUnit`** — Layer 1.5 check re-reads the sheet to verify unit availability before writing
- **Comprehensive documentation** — `DEVELOPING.md` and `TEAM_GUIDELINE.md` are exceptionally well-written
